# Security Policy

## Reporting a Vulnerability

If you discover a security issue in Lucid Visualization Suite, please do not
open a public issue with exploit details.

- Prefer GitHub's private vulnerability reporting flow if it is enabled for the
  repository.
- If private reporting is not available yet, open a minimal public issue asking
  for a secure contact path without including sensitive details, secrets, or
  proof-of-concept exploit steps.

Please include:

- affected version or commit
- impact summary
- reproduction steps
- any suggested mitigation

We will review reports and respond as quickly as practical.
