## Notes on the Critical Fixes

- Core AS export should now preserve actual 3D coordinates and carry adjacency-backed edges into LV-ready sheets.
- Single-step MatrixForge embedding should no longer bypass the AS distance pipeline.
- Dataset ingestion for AS should no longer assume the first sheet defines the full label universe.

## Notes on the High Fixes

- `OptimalChoice` now evaluates every timestep as a candidate anchor, aligns every other timestep against that candidate with Procrustes, sums the residuals, and chooses the lowest-total-residual reference. This replaces the old implicit step-0 fallback.
- Distance-pipeline centrality is now explicit: precomputed-distance runs return `CentralityState::Unavailable` with labels and a reason string instead of zero-filled metrics.
- Generic distance handling now uses `DistanceMatrix`, while `SeMatrix` remains as a compatibility alias for older code and artifacts.
- README and math docs now describe the actual MF similarity -> AS distance bridge and clarify that graph centrality is unavailable for pure distance inputs.
- Integration coverage now includes 3D export fidelity, explicit distance-pipeline centrality behavior, single-output vs single-slice-series bridge equivalence, and `OptimalChoice` execution/label preservation.

## Notes on the Medium and Low Fixes

- The MatrixForge GUI now derives defaults from `MfConfig::default()`, exposes `unicode_normalize`, `shared_vocabulary`, and `min_tokens_per_slice` under Advanced options, and explicitly documents that directory ingestion is non-recursive and `.txt`-only.
- AlignSpace JSON output now includes `procrustes` and `etv_dataset`, so the JSON artifact matches the full in-memory result more closely.
- MF and AS now use weighted shortest paths consistently for distance, closeness, and betweenness, with regression tests covering a case where weighted betweenness differs from the old unweighted behavior.
- `MdsDimMode::Visual` is now documented as the legacy public name for the current 2D planar mode, and the GUI label is simplified to `2D`.
- `SimToDistMethod` now lives in `lv-data` and is re-exported from there, removing MF's dependency on an AS-owned bridge type.
